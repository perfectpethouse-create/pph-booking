# คู่มือติดตั้ง — ระบบจองห้องพัก Perfect Pet House

เว็บแอปนี้ทำงาน 2 โหมด:
- **โหมดทดลอง (mock)** — เปิดใช้ได้ทันที ข้อมูลเก็บในเบราว์เซอร์เครื่องเดียว (ไว้ลองก่อน)
- **โหมดจริง (Firebase)** — ข้อมูลอยู่บนคลาวด์ ซิงค์หลายเครื่อง (Mac + มือถือ) เห็นตรงกัน

ตอนนี้ยังไม่ได้ตั้ง Firebase → แอปจะรันโหมดทดลองให้เอง เมื่อทำตามขั้นตอนด้านล่างเสร็จจะสลับเป็นโหมดจริงอัตโนมัติ

---

## ลองใช้เดี๋ยวนี้ (โหมดทดลอง)

1. เปิดไฟล์ `index.html` ด้วยเบราว์เซอร์ **ผ่านเซิร์ฟเวอร์เล็ก ๆ** (โมดูล JS ต้องรันผ่าน http ไม่ใช่ดับเบิลคลิกไฟล์ตรง ๆ)
   - วิธีง่ายสุดบน Mac: เปิด Terminal ไปที่โฟลเดอร์ `booking-app` แล้วพิมพ์
     ```
     python3 -m http.server 8080
     ```
     จากนั้นเปิดเบราว์เซอร์ไปที่ `http://localhost:8080`
2. หน้าล็อกอิน: โหมดทดลองพิมพ์อีเมล/รหัสอะไรก็เข้าได้
3. ทดลองเพิ่มการจอง สร้างการ์ดส่งลูกค้า ฯลฯ ได้เต็มที่

---

## ตั้งค่าใช้งานจริง (Firebase) — ทำครั้งเดียว ~10–15 นาที

> ผมสร้างบัญชี/โปรเจกต์บนคลาวด์แทนคุณไม่ได้ (เหตุผลด้านความปลอดภัย) — ทำตามทีละขั้นนี้

### 1) สร้างโปรเจกต์ Firebase
1. ไปที่ https://console.firebase.google.com → ล็อกอินด้วย Google ของร้าน
2. กด **Add project** ตั้งชื่อ เช่น `pph-booking` → กดต่อไปจนเสร็จ (ปิด Google Analytics ได้)

### 2) เปิด Firestore Database
1. เมนูซ้าย → **Build → Firestore Database → Create database**
2. เลือก **Start in production mode** → เลือก location `asia-southeast1 (Singapore)` → Enable

### 3) วางกฎความปลอดภัย
1. ในหน้า Firestore → แท็บ **Rules**
2. ลบของเดิม วางเนื้อหาจากไฟล์ `firestore.rules` (ในโฟลเดอร์นี้) แล้วกด **Publish**

### 4) เปิดระบบล็อกอิน + เพิ่มบัญชีพนักงาน
1. เมนูซ้าย → **Build → Authentication → Get started**
2. แท็บ **Sign-in method** → เปิด **Email/Password** → Save
3. แท็บ **Users → Add user** → ใส่อีเมล+รหัสของพนักงานแต่ละคน (เพิ่มได้หลายคน)

### 5) เอา config มาใส่ในโค้ด
1. ไอคอนเฟือง (มุมซ้ายบน) → **Project settings**
2. เลื่อนลงหัวข้อ **Your apps** → กดไอคอน **Web `</>`** → ตั้งชื่อเล่น → Register app
3. จะเห็นบล็อก `const firebaseConfig = { ... }` → คัดลอกค่าข้างในมาวางแทนใน
   ไฟล์ `js/firebase-config.js` (แทนที่ค่า `YOUR_...` ทั้งหมด)

### 6) อัปโหลดขึ้น GitHub Pages
1. สร้าง repo ใหม่บน GitHub → อัปโหลด/พุชทุกไฟล์ในโฟลเดอร์ `booking-app`
2. ไปที่ repo → **Settings → Pages** → Source เลือก branch `main` โฟลเดอร์ `/root` → Save
3. รอสักครู่จะได้ลิงก์ เช่น `https://ชื่อคุณ.github.io/ชื่อ-repo/`

### 7) อนุญาตโดเมนให้ล็อกอินได้
1. กลับไป Firebase → **Authentication → Settings → Authorized domains → Add domain**
2. ใส่โดเมน GitHub Pages ของคุณ (เช่น `ชื่อคุณ.github.io`) → Add

เสร็จแล้ว! เปิดลิงก์ GitHub Pages → ล็อกอินด้วยบัญชีที่สร้างไว้ → ข้อมูลจะซิงค์ทุกเครื่องอัตโนมัติ

## หมายเหตุ
- ต้องการกลับไปโหมดทดลองชั่วคราว: เติม `?mock=1` ท้าย URL
- ค่าใช้จ่าย: แผนฟรี (Spark) ของ Firebase เพียงพอสำหรับร้านขนาดนี้ และไม่พักโปรเจกต์
- อย่าลืมกด **สำรองข้อมูล** เป็นระยะ (เมนูซ้าย 💾) เก็บไฟล์ไว้ที่ปลอดภัย
- ราคาห้อง/จำนวนห้อง/บริการเสริม แก้ได้เองที่เมนู **ตั้งค่า** ไม่ต้องแก้โค้ด
